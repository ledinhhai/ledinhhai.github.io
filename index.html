Tự động deploy dự án của bạn với Circle Ci và Github

Quy trình

Giả sử bạn có 1 dự án viết 1 trang web (trong ví dụ này web được viết bằng .Net Core)
Code của trang web được lưu trữ ở github và được deploy trên Server.
Khi bạn có thay đổi trên dự án và được lưu trữ lên github thì circle ci sẽ lắng nghe những thay đổi đó sao đó gọi server lấy code mới về deploy.

Yêu cầu

  1. Có project được lưu trữ sẵn trên github
  2. Server có setup sẵn ssh hỗ trợ connect qua public key và private key

Cài đặt server

  1. Server cần clone source từ GitHub bằng ssh
  2. Cần viết 1 script pull code và deploy site

Cài đặt Circle CI

  1. Vào trang https://circleci.com/vcs-authorize/ đăng nhập Circle CI bằng github
  2. Chọn project bạn muốn cài đặt.
    Click button Set Up Project để cài đặt cho project bạn muốn cài đặt Chọn Use Existing Config Chọn Start Buiding
  3. Thêm SSH Private key của server cho Circle CI. Tham khảo: https://circleci.com/docs/2.0/add-ssh-key/#:~:text=In%20the%20CircleCI%20application%2C%20go,the%20Add%20SSH%20Key%20button.

    Cài đặt Source Project

  4. Trong thư mục gốc của project tạo thư mục .circleci và tạo thêm file config.yml bên trong đó.
    Tham khảo config https://github.com/ledinhhai/CicrleCiDemo/blob/master/.circleci/config.yml

Bundling và Minification Trong DotNet Core

Bundling là gì

Hiểu đơn giản bundling có nghĩa là gộp nhiều file (css, js) thành 1 file duy nhất. Việc này giúp giảm số lượng request cần thiết lên server để hiện thị nội dung của trang web.
Ít file hơi đồng nghĩa với việc ít HTTP request từ trình duyệt tới sever giúp cải thiện hiệu suất tải trang.

Minification là gì

Minification giúp bạn giảm bỏ các ký tự không cần thiết khỏi mã code mà không thay đổi chức năng của nó. Điều này làm giảm đáng kể kích thước của tài nguyên (css, js, image..) được yêu cầu.
Minification sẽ rút ngắn các tên biến xóa các comment và khoảng trắng k cần thiết.

Không sử dụng B/M

Sử dụng B/M

Cài đặt Bundling và Minification trong DotNet Core

Cài đặt Package BuildBundlerMinifier
Tại file bundleconfig.json

Nội dung mẫu cho file bundleconfig

outputFileName: đường dẫn file sau khi gộp.
inputFiles: là mảng cái file cần gộp (ngăn cách bởi dấu phẩy).
có thể sài pattern cho inputFiles. Ví dụ “inputFiles”: [“wwwroot/*/!(.min).css” ] sẽ gom tất cả file css không bao gồm file đã gộp (có đôi .min)

Tham khảo thêm tại: https://docs.microsoft.com/en-us/aspnet/core/client-side/bundling-and-minification?view=aspnetcore-3.1&tabs=visual-studio
